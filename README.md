# ✨AstroAgent

An AI-powered astrology companion built with **LangGraph, FastAPI, React, Swiss Ephemeris, and Gemini**.

AstroAgent helps users explore their birth chart, understand daily planetary transits, and receive grounded astrological guidance through a conversational interface.

---

## Overview

This project was built as part of the **Aradhana AstroAgent Assignment**.

The goal was to create an agentic astrology assistant that:

* Uses real astrological data
* Calls tools when needed
* Reasons through a graph-based workflow
* Provides warm and thoughtful responses
* Includes a reproducible evaluation framework

---

## Features

###  Birth Chart Analysis

Uses Swiss Ephemeris to calculate:

* Sun
* Moon
* Mercury
* Venus
* Mars
* Jupiter
* Saturn
* Rahu
* Ketu
* Ascendant (Lagna)

Planetary positions are computed from real astronomical calculations rather than being generated by the language model.

---

### Daily Transit Analysis

Computes current planetary positions and generates transit-based readings in relation to the user's natal chart.

---


### Persistent Memory

Birth details are stored locally and automatically reused across sessions.

Users do not need to repeatedly enter:

* Date of birth
* Time of birth
* Place of birth

---

### Chart Caching

Previously computed charts are cached to avoid repeated calculations.

Benefits:

* Faster responses
* Reduced computation
* Lower latency

---
### 📚 Retrieval-Augmented Generation (RAG)

AstroAgent uses a curated astrology knowledge base to ground interpretations.

Benefits:

* More consistent readings
* Reduced hallucinations
* Better alignment with astrology reference material

## Architecture

### LangGraph Workflow

```text
User
 ↓
Router Node
 ↓
 ┌───────────────┐
 │ Birth Chart   │
 │ Transit Route │
 │ General Route │
 └───────┬───────┘
         ↓
 Tool Execution
         ↓
 Knowledge Lookup (RAG)
         ↓
 Gemini
         ↓
 Final Response
```

### Graph Nodes

#### Router Node

Classifies requests into:

* Birth chart questions
* Daily transit questions
* General questions

#### Tool Node

Computes birth charts using:

* Swiss Ephemeris
* Geocoding data

#### Transit Node

Computes daily transits and relates them to the natal chart.

#### Interpretation Node

Combines:

* Tool outputs
* Retrieved astrology knowledge
* User question

to generate the final response.

## Implemented Tools

### 1. compute_birth_chart()

Returns:

* Planetary positions
* Zodiac signs
* Ascendant

Uses Swiss Ephemeris for real astronomical calculations.

---

### 2. compute_daily_transits()

Calculates current planetary positions and produces transit readings.

---

### 3. geocode_place()

Converts place names into:

* Latitude
* Longitude

Required for accurate astrological calculations.

---

### 4. knowledge_lookup()

Retrieves relevant astrology reference notes using RAG.

## Evaluation Framework

Evaluation was treated as a first-class deliverable.

A versioned JSONL dataset containing:

* Birth chart requests
* Transit requests
* Ambiguous prompts
* General questions
* Failure cases

## Safety Guardrails

AstroAgent does not:

* Provide medical certainty
* Provide legal certainty
* Provide financial certainty
* Claim astrological predictions as facts

Responses are framed as guidance and reflection rather than guarantees.

## Known Limitations

* Houses are not yet fully utilized in interpretations.
* Transit analysis currently focuses on planetary positions rather than advanced aspect calculations.
* Streaming responses are not yet implemented.
* Memory currently uses local storage rather than a database-backed user system.
